# Code of Conduct

Lictor is built for people who don't have a security team. That includes a lot of people who got told "security is too complicated for you" by people who profited from that being true. We don't repeat that pattern here.

## Three rules

1. **Make people smarter about their own code, not stupider.** Explain things. Cite specifics. If you know something a contributor doesn't, share it without making them feel small.

2. **Be specific.** "This looks wrong" is not a code review. "Line 47 dereferences a value that can be null on the failed path" is. Apply the same rule to bug reports: file paths, line numbers, repro steps.

3. **Don't moralize.** Vibe-coding, weekend SaaS, AI-generated code — those aren't ethical failures. They're how a generation of new builders is shipping software. Our job is to help them ship it safely, not to lecture them.

## What gets you removed

- **Personal attacks** on contributors. Disagree with the code, not the person.
- **Discrimination** on any axis. Race, gender, sexuality, religion, nationality, age, neurotype, professional background. We're not interested.
- **Bad-faith engagement.** Repeated dismissive comments, troll behavior, derailing threads.
- **Disclosing vulnerabilities publicly** before our [security policy's](./SECURITY.md) coordinated disclosure window. This is a real harm; we treat it like one.
- **Cherry-picked screenshots** used to embarrass anyone — contributor, user, or third party. This is a security project; we hold ourselves to a higher bar on responsible communication.

## How to report

- **General CoC issues:** open a private GitHub Security Advisory at [github.com/Raffa-jarrl/Lictor-AI/security/advisories/new](https://github.com/Raffa-jarrl/Lictor-AI/security/advisories/new) — yes, the same channel as security bugs, because the maintainer who reads those is the same person who'd handle a CoC report
- **Direct contact:** [hello@lictorai.com](mailto:hello@lictorai.com)
- **Anonymous tip:** the GitHub Security Advisory form supports anonymous submissions

## What we do with reports

1. **Acknowledge within 24 hours.** Even if we're traveling. Even if we don't have a substantive response yet.
2. **Investigate quietly.** We don't broadcast that an investigation is happening. The reporter's identity is protected.
3. **Act.** Outcomes range from a private clarifying message to a permanent ban from all Lictor spaces. We document every decision.
4. **Tell you what we did.** The reporter hears back. The reported party hears back. Other contributors don't unless the situation requires it.

## A note on the "Mythical Reasonable Person"

Some communities define their CoC around "reasonable behavior." That standard fails when the people defining "reasonable" share a background. Our standard is closer to: **"Would this make a new contributor — different from the people currently in the room — feel that this is a place where their code can succeed on its merits?"** When the answer is no, we adjust the room, not the new contributor.

— The Lictor maintainers
