# Lictor — Marketing Performance Report Template

> **Purpose:** zero-friction weekly / monthly / quarterly cadence that Reel + Bridge + Conductor can populate without Dor lifting a pen.
> **Stack assumption:** GitHub (traffic), Buttondown (Beacon waitlist), Twitter/X (founder channel), YouTube (teardowns), HN/Reddit (launch surfaces). No paid spend in Year 1.
> **Status:** dormant until **Day +0 = 2026-10-06** (public launch). Pre-launch we only run the **GitHub-only baseline** version.

---

## Cadence map

| Cadence | When it runs | Who writes it | Who reads it | Format |
|---|---|---|---|---|
| **Daily pulse** | Every morning 09:00 IST | Conductor (auto-pulled, no commentary) | Dor only | One-line JSON in `output/metrics/daily.jsonl` |
| **Weekly review** | Fridays 14:00 IST | Reel (drafts) + Mirror (reviews) | Dor + public newsletter footer | Markdown at `output/reports/weekly/YYYY-Www.md` |
| **Monthly report** | 1st of month, 06:00 IST | Conductor (drafts) + Dor (edits) | Public (publishes to lictor.ai/transparency) | Markdown + 1 chart |
| **Quarterly business review** | Last Friday of quarter | Dor (full rewrite, agents prep data) | Internal + future investors/acquirers | Slides at `docs/business/qbr/YYYY-QN.pptx` |

The point of the cadence ladder: the daily/weekly pulses are cheap (auto) — the monthly/QBR is where Dor actually thinks.

## METRIC LADDER (the only thing that matters)

Three tiers. Every metric in the reports below maps to exactly one tier.

### Tier 1 — North-star metrics (3 only)

These are the numbers that decide whether Lictor matters. Everything else is leading indicator.

| Metric | Source | Target by 2027-05-15 (Y1 end) | Why this one |
|---|---|---|---|
| **Weekly active installs** (people running `/lictor-security-check` at least once in last 7d) | Anonymous opt-in ping from CLI (default off — only counted if user opts in) | 2,500 WAI | The only number that proves the tool gets used, not just starred |
| **Audit-as-a-Service revenue (MRR-equivalent)** | Stripe + Paddle | $15k MRR-eq by Q1'27, $25–35k by Q2'27 | The thing that funds Year 2; the only paid surface in Year 1 |
| **GitHub stars** | `gh api repos/Raffa-jarrl/Lictor-AI` | 10,000 stars | The acquirer signal + the OSS-credibility signal |

### Tier 2 — Funnel metrics

Inputs to Tier 1. Track weekly; act when one stalls.

| Funnel stage | Metric | Source | Healthy weekly Δ post-launch |
|---|---|---|---|
| Awareness | Unique repo visitors / week | GitHub Traffic API (`/traffic/views`) | +15% wow (first 8 weeks), then +3-5% wow |
| Awareness | Twitter/X founder follower count | manual / Twitter API | +50/wk (organic) |
| Awareness | Teardown video views | YouTube Data API (already wired) | +30% wow per upload, with retention >35% |
| Interest | Beacon waitlist signups | Buttondown API | 50/wk pre-launch, 200/wk launch week, 30/wk steady state |
| Interest | Newsletter open rate | Buttondown | >40% (we send infrequent, audience is warm) |
| Activation | Repo clones (last 14d, unique cloners) | GitHub `/traffic/clones` | +10% wow |
| Activation | First-`/lictor-security-check`-run events | CLI opt-in ping | match clone count within ±20% — gap = install friction |
| Retention | Returning installers (running ≥2x in 14d) | CLI ping | >35% of first-runners |
| Revenue | AaaS demo calls booked | Calendly | 4/wk by Q1'27 |
| Revenue | AaaS demo→close rate | manual / Stripe | >30% (it's a niche audience; if lower, pricing is wrong) |

### Tier 3 — Engineering / health metrics

Not strictly marketing but they leak into perception. Include in monthly + QBR only.

| Metric | Source | Target |
|---|---|---|
| CI green rate (last 30d) | `gh api .../actions/runs` | >90% |
| Time-to-first-response on GitHub issues | Bridge logs | <24h for non-security, <2h for `false-negative`-labeled |
| `false-negative` issue→fix lead time | Bridge + dev logs | <14d median |
| Open security issues count | GitHub Security tab | 0 critical/high |

## QUARTERLY BUSINESS REVIEW (QBR) — `docs/business/qbr/YYYY-QN.pptx`

Different format. Slides, not markdown. Internal-only until something needs to leave the building (investor / acquirer / hire).

### Slide structure (~15 slides)

1. **Title** — Quarter, headline, the one sentence.
2. **Tier-1 scoreboard** — 3 north-star metrics, this Q vs last Q vs Y1 target.
3. **What changed this quarter** — bullets: launches, hires, churns, pivots.
4. **Funnel — awareness** — chart of unique visitors + star growth + Twitter followers.
5. **Funnel — interest** — Beacon waitlist + newsletter health + top inbound channels.
6. **Funnel — activation/retention** — install→run→return funnel.
7. **Revenue** — AaaS pipeline, won/lost/pending, MRR-eq trajectory, gross margin.
8. **What worked** — top 3 plays with the data that backs them.
9. **What didn't work** — top 3 misses with honest hypotheses.
10. **Cohort retention** — first-time installers from each month of this Q, returning %.
11. **Competitive watch** — Probe agent summary of competitors' moves this Q.
12. **Engineering health** — CI green rate, time-to-fix, security issue trends.
13. **Next quarter — strategic theme** — one theme, three sub-bets, with success criteria.
14. **Next quarter — KPIs** — explicit numerical targets.
15. **Asks** — what Dor needs (decisions, hires, budget approvals — even for a one-person shop, externalize these).

**Audience:** if Dor only. If for an investor / acquirer pitch, slot in market-size + acquirer-strategic-fit slides between #11 and #12 (use the existing `docs/launch/acquirer-deck-template.md`).

## ANTI-PATTERNS — things to never let creep in

1. **Vanity totals.** "Total stars since inception" is fine for the README, not for performance review. We track weekly Δ.
2. **Aggregating across funnel stages.** "Total community size" smushes visitors + waitlist + installers + paying customers into one meaningless number.
3. **Tracking metrics we won't act on.** If a number can't trigger a decision, it doesn't belong in the report.
4. **Optimizing for the dashboard.** If WAI is the metric, don't game it by re-pinging from the same machine. Tier 1 metrics need adversarial review — Mirror's job.
5. **Slipping the cadence.** Monthly transparency report on the 1st, no exceptions. A missed publish is a brand-trust event we can't recover from quickly.
