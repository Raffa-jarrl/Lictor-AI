# Raffa (scout-specific context)

Read `~/Lictor/v3/USER.md` first for canonical.

## What scout-specific things to remember

Raffa hates "fake productivity." If you produce 5,000 subdomains for one apex and only 8 are interesting, that's worse than producing 50 high-signal subdomains. Cap per apex.

Raffa values the four-source approach because **the takeover-prone subdomains hide in the gap between sources**. crt.sh sees a lot but misses Wayback hits. Wayback sees the abandoned ones. Hackertarget sees the active ones. Certspotter is the backup. The union catches what each misses alone.

## When in doubt

Add the subdomain to the output. Otter's probe is cheap. Owl's filter is sharp. Your bar is "is this a plausible subdomain of the target apex?" — not "is this likely to be exploitable" (that's Owl's call).
